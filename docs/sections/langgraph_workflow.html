LangGraph Workflow

From user input to assistant reply, four LangGraph nodes coordinate LLM calls, parsing, validation, and state updates.

Node Timeline

generate – call_*_model

  • Compose system instructions and conversation history.
  • Emit llm_request and llm_response telemetry.
  • Return {"raw_response": ...}.

parse – coerce_to_schema

  • Extract JSON from the raw completion.
  • Normalise disease names, yes/no answers, medication free text.
  • On failure, log parse_error and skip updates.

validate – run_disease_validation

  • Link diseases via RapidFuzz; populate linked/unknown/generic lists.
  • Perform SHACL specificity check (or the child-map shortcut).
  • Emit validation_result* events with outcomes.

end – merge

  • LangGraph END node returns the diff to merge into session_state.
  • Streamlit renders the assistant prompt + structured panel.
  • Option shortcuts persist for quick numeric replies.

Practical Example

# inside frontend/app.py
raw_message = "type 2 diabetes mellitus"
canonical = resolve_option_shortcut(raw_message)
add_message("user", canonical, display_content=f"{raw_message} ({canonical})")
result = graph.invoke({"user_messages": [m["content"] for m in st.session_state.messages if m["role"] == "user"]})
update_structured_data(result.get("structured"))
assistant_reply = build_assistant_reply(st.session_state.structured_data)
add_message("assistant", assistant_reply)

The same workflow can be reused in automated tests by supplying canned user message histories and inspecting the resulting state diff.
