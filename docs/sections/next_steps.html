Suggested Next Steps

Prioritised ideas to harden the product, expand coverage, and reduce manual workload for pharma teams.

Roadmap Themes

High impact

Automated option shortcuts

Teach the parser to recognise “option 4” even if the user refers to a prior turn, ensuring hands-free refinement during live workshops.

Medium impact

Therapeutic area validation

Add SHACL shapes tying diseases to allowable therapeutic areas, preventing mismatched TA selections in exports.

High impact

Incremental ontology updates

Integrate ROBOT or Ontofox to diff weekly DOID releases and regenerate the label index without manual steps.

Operational Enhancements

  • Introduce Prometheus metrics (LLM latency, validation latency, unknown term count) for live dashboards.
  • Automate daily smoke tests via CI so ontology or prompt regressions are caught before users notice.
  • Seed the exported JSON into downstream study trackers to prove end-to-end integration.

User Experience Refinements

  • Add a “try another disease” button whenever suggestions exhaust without a confident match.
  • Surface the ontology label and DOID on hover inside the chat bubbles for transparency.
  • Offer CSV/PDF export for stakeholders that cannot consume JSON directly.
