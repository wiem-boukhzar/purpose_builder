System Landscape

An expanded look at the actors and layers that make up the Purpose Creator deployment.

Stakeholders

The agent targets two primary audiences:

  • Discovery / clinical researchers capture high-level intents (disease focus, medication, collaborators, product development) in a conversational manner.
  • Regulatory and portfolio teams consume the exported JSON/validation report to seed downstream systems or track portfolio health.
Example workflow: A translational scientist sketches a new program targeting “type 2 diabetes mellitus” and “obesity”. The agent ensures each disease maps to a DOID and prompts for therapeutic area, medication plans, collaborators, and product development status.

Layer Overview

Streamlit UI

  • Chat-driven data collection with numbered suggestion shortcuts (e.g., responding “option 3”).
  • Real-time structured summary card showing validation messages and download options.

Purpose Application Layer

  • LangGraph workflow coordinates LLM extraction, parsing, and validation.
  • Logging utilities output JSON events for chat turns, LLM calls, and validation results (purpose_app/logging_utils.py).

Ontology & Validation Services

  • Loads the Disease Ontology snapshot, builds label/synonym indexes, and precomputes subclass hints.
  • Supports both full SHACL specificity checks and the performance-oriented child-map shortcut.

Data & Infrastructure

  • data/ontology/doid.owl anchors validation to a known DO release.
  • ontology_validation/shapes.ttl encodes SHACL constraints so auditors can review them.
  • Environment variables control which LLM endpoint is used and where logs are written.
