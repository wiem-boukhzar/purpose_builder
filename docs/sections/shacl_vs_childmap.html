SHACL vs. Pre-computed Child Map

Two interchangeable validators keep diseases specific. SHACL offers standards-based rigor, while the child-map shortcut prioritises latency.

Decision Matrix

Dimension SHACL (pySHACL) Child-map Validator
Latency ~450–600 ms per run (RDFS inference + SPARQL) <20 ms per run (dict lookups)
Extensibility Declarative – add new constraints via Turtle shapes
Audit trail Full SHACL report graph persisted to disk Logged JSON events only (no RDF artefact)
Runtime footprint Higher CPU and memory bursts during validation Minimal additional load once child map is built
Best suited for Regulated environments, compliance reviews Demo booths, call-centre accelerators, low-power hosts

How Each Path Works

SHACL Path

Pipeline

  • validate_diseases() builds an RDF data graph.
  • pyshacl.validate(..., inference="rdfs") evaluates shapes.
  • Violations return focus nodes; we re-query subclasses for suggestions.

Practical example

report = validate_diseases([iri], MONDO_GRAPH)
if not report.conforms:
    for generic in report.generic_concepts:
        options = suggest_children(generic, MONDO_GRAPH)

Resulting SHACL TTL is persisted for audit together with the final purpose JSON.

Pre-computed Path

Pipeline

  • build_child_map() records every class with subclasses.
  • check_specificity() tests membership in HAS_CHILDREN.
  • Suggestions come from the same cached adjacency list.

Practical example

linked = link_many(["diabetes mellitus"], LABEL_INDEX, MONDO_GRAPH) report = validate_diseases([ld.iri for ld in linked], MONDO_GRAPH) assert not report.conforms assert report.generic_concepts == ["http://purl.obolibrary.org/obo/DOID_9352"]

When to Choose Which

Recommendation: default to SHACL for enterprise deployments where auditability and declarative governance matter. Switch to the child-map validator only when response time is the top priority and downstream consumers accept JSON logs as evidence.
  1. Pilot / demo – enable the child map to keep the chat snappy while stakeholders explore scenarios.
  2. Production – retain SHACL to generate compliance-grade validation reports alongside the exported purpose JSON.
  3. Hybrid – some teams run child map during ideation and toggle SHACL on right before submitting the final purpose artifact.
