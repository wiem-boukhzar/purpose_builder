Ontology Data Flow

Walkthrough of what happens from the moment a disease string enters the system until validation finishes.

Step-by-step

  1. Load Disease Ontology – load_graph() parses doid.owl, caching the RDF graph (approx. 15k classes).
  2. Build indexes – build_label_index normalises labels; build_child_map notes subclass relationships.
  3. Fuzzy link – link_many compares the user string against the index using RapidFuzz token-set ratio.
  4. Specificity check – SHACL (validate_diseases) or child-map lookup ensures the chosen class is not an umbrella concept.
  5. Suggestions – If the concept is generic, suggest_children enumerates direct subclasses for refinement.

Example

# user enters "NSCLC"
results = link_many(["NSCLC"], LABEL_INDEX, MONDO_GRAPH)
linked = results[0]
assert linked.label == "lung non-small cell carcinoma"
report = validate_diseases([linked.iri], MONDO_GRAPH)
assert report.conforms  # concept has no immediate subclasses
