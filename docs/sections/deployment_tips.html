Deployment Tips

Whether you ship to Streamlit Cloud, an internal VM, or a VDI lab, these patterns keep the purpose agent reliable.

Streamlit Cloud / PaaS

  • Bundle data/ontology/doid.owl in the repo; Streamlit caches the 20 MB file but cold starts take ~8 s.
  • Use requirements.txt; avoid system packages that the managed runtime cannot install.
  • Store secrets via st.secrets; rotate API keys quarterly.

Enterprise VM / Docker

Container recipe

FROM python:3.10-slim
WORKDIR /app
COPY . /app
RUN python -m venv /opt/venv \
 && /opt/venv/bin/pip install --upgrade pip \
 && /opt/venv/bin/pip install -r requirements.txt
ENV PATH="/opt/venv/bin:$PATH"
ENV STREAMLIT_SERVER_PORT=8501
CMD ["python", "-m", "streamlit", "run", "frontend/app.py"]

Networking

  • Whitelist outbound HTTPS to the Apollo/OpenAI-compatible endpoint.
  • Lock down inbound traffic to the Streamlit port via security groups.

Authentication

  • Pair Streamlit with corporate SSO (e.g., Azure AD) using reverse proxies such as Nginx + Auth0 or Azure Front Door.
  • If embedding into an internal portal, expose the LangGraph workflow via a FastAPI microservice and keep Streamlit for admin tooling.

Runtime switching

  • Record the chosen mode in logs/purpose_app.log so audits know which validator was active.
