Ontology & Validation Layer

Components under ontology_validation/ that power disease linking, specificity validation, and refinement suggestions.

Modules

ontology_loader.py

  • load_graph(path="data/ontology/doid.owl") parses the DO snapshot (cached via lru_cache).
  • build_label_index collects rdfs:label, skos:altLabel, and oboInOwl:hasExactSynonym entries (English/default).
  • build_child_map pre-computes subclass relationships (HAS_CHILDREN) reused by the child-map validator.

disease_linker.py

  • link_best normalises input (ASCII fold, punctuation removal) and selects the highest-ranking label via RapidFuzz.
  • link_many applies link_best to a list of terms, returning LinkedDisease or UnknownDisease.
  • label_for retrieves prettified labels; results are cached for performance.

validator.py

  • Creates an RDF data graph with a single ex:ResearchIntent node.
  • Runs pyshacl.validate with ontology_validation/shapes.ttl to enforce specificity.
  • Returns ValidationReport containing conforms, generic_concepts, and report_text.

suggestions.py

  • suggest_children(iri) enumerates direct subclasses (English labels) used to prompt for more specific diseases.

Key Assets

  • data/ontology/doid.owl – pinned Disease Ontology snapshot (~26 MB in RDF/XML).
  • ontology_validation/shapes.ttl – SHACL shapes enforcing “at least one disease” and “no umbrella classes”.
