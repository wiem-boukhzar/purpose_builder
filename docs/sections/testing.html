Testing Strategy

Regression tests lean on the real ontology snapshot so that fuzzy matching, SHACL validation, and shortcut behaviours stay trustworthy.

Test Inventory

File Focus Key assertions
tests/test_linker.py RapidFuzz matching “NSCLC” resolves to a non-small cell carcinoma label.
tests/test_validator.py Specificity checking Diabetes mellitus fails SHACL; type 2 passes after refinement.
tests/test_suggestions.py Subclass coverage Umbrella IRIs surface expected children (type 1/2, gestational, etc.).
tests/test_smoke.py Conversation flow Multiple diseases, misspellings, and numeric shortcuts stay coherent.

Running the Suite

python -m venv .venv
source .venv/bin/activate
python -m pytest -q
Tests read the bundled data/ontology/doid.owl; ensure the file is present locally. Runtime ~15 s on a modern laptop.

What to Add Next

Option shortcuts

Create a fixture with numeric replies (“option 4”) to ensure resolve_option_shortcut keeps the state aligned with the displayed labels.

Final validation reports

Assert that run_final_pyshacl_validation produces a conforming payload before the JSON download is enabled.

Telemetry assertions

Patch logging_utils.log_event in tests to confirm that each LangGraph step emits the expected JSON payloads.

Sample Failure Diagnosis

E   AssertionError: Expected 'type 2 diabetes mellitus' but got 'diabetes mellitus'

Re-run with: pytest tests/test_validator.py -k type_2
Inspect: logs/purpose_app.log  # validation_result payload contains offending IRIs

Because tests use real ontology data, a failing fuzzy match often indicates the label index was rebuilt or the ontology snapshot changed. Re-run build_label_index manually to inspect available synonyms before adjusting thresholds.
