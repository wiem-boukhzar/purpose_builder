Telemetry & Logging

All runtime events are recorded through purpose_app/logging_utils.py, enabling enterprise-grade observability.

Logger Configuration

The logger initialises a RotatingFileHandler writing JSON lines to logs/purpose_app.log (override with PURPOSE_LOG_PATH). Each entry includes timestamp, event, and custom payload.

Event Catalogue

EventPayloadTriggered by
chat_message{"role": "user", "content": "type 2 diabetes", "display": "option 5 (type 2 diabetes)"}add_message
llm_request{"provider": "apollo", "model": "gpt-4o-mini", "message_count": 5}call_local_model
llm_response{"status": "ok"} or {"status": "error", "error": "Timeout"}LLM wrapper
