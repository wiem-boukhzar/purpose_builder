Operational Guidance

Keep the agent healthy in day-to-day operations: manage environments, watch telemetry, and know when to switch validation modes.

Platform Checklist

Environment hygiene

  • Create and activate a fresh Python environment (e.g., python -m venv .venv); pin rapidfuzz, rdflib, and pyshacl.
  • Store secrets with Streamlit’s st.secrets or environment variables; avoid embedding API keys in code.

Ontology assets

  • Confirm data/ontology/doid.owl matches the version string exported in final JSON.
  • Update the SHA/version in release notes if the ontology snapshot changes.

Telemetry

  • purpose_app/logging_utils.py writes JSON lines to logs/purpose_app.log by default.
  • Ship the log directory to ELK, Splunk, or Azure Monitor for central dashboards.

Key Operational Playbooks

Latency spikes

  • Inspect validation_result events; if final SHACL runs dominate latency, confirm users are not repeatedly triggering the “Run Final MONDO Validation” button.
  • Cache LangGraph results per message when demoing (Streamlit reruns can repeat validation otherwise).

Ontology mismatches

  • Users reporting “wrong subtype” likely selected a generic concept; confirm suggestions include the expected child.
  • Use docs/SHACL_vs_Precomputed.md to brief stakeholders on trade-offs before changing validation modes.

Governance & export

  • Persist only the final purpose JSON and the SHACL report (if applicable) per requirements.
  • Review docs/sections/file_reference.html when onboarding new operators so they understand where artefacts live.
