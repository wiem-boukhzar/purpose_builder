Essential Helpers

Reference for the core helper functions used throughout the Purpose Creator workflow.

run_disease_validation

  • Normalises the disease list and calls link_many.
  • Runs SHACL specificity (or child-map) validation, caching results.
  • Writes linked/unknown/generic results back to session_state and emits validation_result logs.
outcome = run_disease_validation([
    "type 2 diabetes mellitus",
    "obesity"
])
assert outcome.structural_fields == "conforms"

build_assistant_reply

  • Checks ontology_generic_diseases and ontology_unknown_diseases to determine if refinements are needed.
  • Populates option_shortcuts so the user can respond with “option 2”.
  • Falls back to prompting the next missing structured field (“Still need: Collaborator, Product Development”).

build_purpose_payload

  • Creates the JSON exported to file (diseases with DOIDs/IRIs, TA, medication, etc.).
  • Includes ontology metadata ({"name": "DO", "version": MONDO_VERSION}).
payload = build_purpose_payload(st.session_state.structured_data)
with open("research_concept.json", "w") as f:
    json.dump(payload, f, indent=2)

Helper Utilities

  • resolve_option_shortcut – maps “option 3” to the canonical disease label.
  • update_structured_data – merges the latest structured fields while preserving existing answers.
  • format_numbered_list – generates the human-readable numbered suggestions surfaced in the chat.
